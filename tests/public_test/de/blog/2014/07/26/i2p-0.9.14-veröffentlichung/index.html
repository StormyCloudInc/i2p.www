I2P 0.9.14 Veröffentlichung

By zzz

0.9.14 enthält kritische Korrekturen für XSS- und Remote-Codeausführungs-Schwachstellen, die von Exodus Intelligence gemeldet wurden. Als zusätzliche Vorsichtsmaßnahme haben wir mehrere erweiterte Konfigurationsfunktionen in der router-Konsole deaktiviert, einschließlich der Installation neuer Plugins. Wir planen, diese nach weiterer Prüfung in einer zukünftigen Version wieder zu aktivieren.

Aufgrund von Änderungen an den I2P-Bibliotheken müssen I2P-Bote-Benutzer ihr Plugin auf Version 0.2.10 aktualisieren, um mit I2P 0.9.14 kompatibel zu sein. Ihr router sollte das Plugin automatisch aktualisieren, nachdem der router neu gestartet wurde.

Die Veröffentlichung enthält außerdem mehrere Fehlerbehebungen in i2ptunnel, i2psnark und anderen Bereichen sowie Aktualisierungen auf die neuesten Jetty-, Tomcat- und Wrapper-Versionen. Wir haben außerdem eine schnellere und sicherere Methode für das Reseeding implementiert. Natürlich gibt es auch die üblichen kleineren Fehlerbehebungen und Aktualisierungen der Übersetzungen.

Sie müssen sofort auf diese Version aktualisieren. Die beste Möglichkeit, die Sicherheit aufrechtzuerhalten und dem Netzwerk zu helfen, ist, die neueste Version auszuführen.

DETAILS ZUR VERÖFFENTLICHUNG

Security Fixes

  • Fix several XSS issues
  • Disable changing news feed URL from UI
  • Disable plugin install
  • Disable setting unsigned update URL from UI
  • Disable clients.config editing from the UI
  • Add Content-Security-Policy and X-XSS-Protection headers
  • Disable unused ExecNamingService (thx joernchen of Phenoelit)

Sicherheitskorrekturen

  • Fix tunnel building so it doesn’t get “stuck” on a single pool
  • Reject participating tunnels when hidden
  • Several i2psnark improvements and fixes (GUI and DHT), including changes for better compatibility with Vuze

Fehlerbehebungen

  • Reseeding now fetches a signed zip file containing router infos for security and speed
  • Use JVM’s AES implementation if it is faster
  • More advanced options shown in the i2ptunnel edit pages
  • Per-message reliabilitiy settings in I2CP and error propagation back from router to client
  • Lots of findbugs fixes and cleanups
  • Support signature types in SAM, bump rev to 3.1
  • New event log page in console
  • Jetty 8.1.15.v20140411
  • Tomcat 6.0.41
  • Wrapper 3.5.25 (new installs and PPA only)
  • Translation updates
  • Update GeoIP data (new installs and PPA only)
